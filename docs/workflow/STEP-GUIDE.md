# STEP Execution Guide

## Purpose

Defines how to execute development work in controlled steps.

---

# STEP Structure

Each feature should follow:

STEP-XX-FEATURE-NAME

Inside each STEP:

1. Context
2. Relevant ADRs
3. Planned Layer Structure
4. Implementation Tasks
5. Acceptance Criteria

---

# Execution Flow

1. Planner defines STEP.
2. Implementer codes.
3. Reviewer verifies compliance.
4. Only then is STEP marked DONE.

---

# No Direct Coding Rule

No feature should be implemented without a defined STEP.
